# Troubleshooting Documentation

This folder contains diagnostic guides, debugging procedures, and solutions for common issues.

## Contents

- **BUSINESS_IMAGES_EDGE_CASES.md** - Edge cases and issues related to business images
- **BUSINESS_IMAGES_EDGE_CASES_FIXES.md** - Fixes for business images edge cases
- **BUSINESS_LOGIC_EDGE_CASES.md** - Edge cases in business logic
- **DEBUG_IMAGE_UPLOAD.md** - Debugging guide for image upload issues
- **DIAGNOSTIC_SIMILAR_BUSINESSES_AND_IMAGES.md** - Diagnostic guide for similar businesses and image issues
- **FIX_RLS_STORAGE_ERROR.md** - Fix for RLS (Row Level Security) storage errors
- **RLS_ANALYSIS_FOR_BUSINESS_IMAGE_UPLOAD.md** - RLS analysis for business image uploads
- **SAVED_BUSINESSES_EDGE_CASES.md** - Edge cases for saved businesses feature
- **TROUBLESHOOTING_SIMILAR_BUSINESSES.md** - Troubleshooting guide for similar businesses feature
- **USER_LOGIC_EDGE_CASES.md** - Edge cases in user logic

## Using This Documentation

When encountering an issue:
1. Search this folder for related documentation
2. Follow the diagnostic steps provided
3. Check for fixes or workarounds
4. If creating new troubleshooting docs, include:
   - Problem description
   - Diagnostic steps
   - Root cause analysis
   - Solution or workaround
   - Prevention measures

## Adding New Troubleshooting Documentation

When documenting a new issue or fix:
1. Create a descriptive filename using `kebab-case` (e.g., `debug-feature-x.md`)
2. Include problem description, symptoms, and diagnostic steps
3. Document the root cause and solution
4. Add prevention measures if applicable
5. Update this README with a brief description
